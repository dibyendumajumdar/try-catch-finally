Introduction to YASE

YASE stands for 'Yet Another Search Engine'. I am looking for a better name for this product - please feel free to send any suggestions.

YASE is essentially a text indexing and retrieval system (search engine).

Main features of YASE

  • YASE is written in ISO C, and is being developed and tested on various flavours of Linux. I have recently ported YASE to Win32.
  • YASE can scan existing directory structures or web sites for documents, therefore it can work with your existing document collections.
  • A single YASE database can contain references to files in multiple directories or web sites.
  • YASE is easy to use - there are only two commands you need to use. One to build a database, the other to query it.
  • YASE can be customised to read any type of document by providing a suitable filter to extract text out of the document.
  • YASE indexes all words in a document. It can optionally "stem" words.
  • YASE is designed to process large collections - attempts are made to compress the YASE database as much as possible.
  • YASE supports searching any or all terms in a query. It also supports partial or exact match of terms.
  • YASE can rank query results by relevance - it uses cosine vector similarity formula for ranking.
  • YASE (as of version 0.9.7) supports boolean queries as well.
  • YASE code quality is reasonably good - better than many competitive products. In particular global variables are avoided.
  • YASE is modular in architecture - modules are fairly independent of each other which means a module can be enhanced easily.
  • YASE is well documented - which should make it easier for you to understand how it works, and enhance it. If anything about YASE is unclear to you, please ask and I will try my best to update the documentation so that your query is answered.
  • YASE is supported by me - in other words, if you find a bug I will try my best to fix it.
  • YASE is Free Software - distributed under GPL. This means you can virtually do whatever you like with it (Freedom) provided you do not deny others the same freedom on any product derived from YASE.

Drawbacks of YASE

YASE is work in progress - therefore many features are planned but have not yet made it. Examples:

  • 64-bit file input/output for YASE database files.
  • Ability to add documents to an existing collection. Currently the database needs to be rebuilt if the document collection changes.
  • Query processing logic can be improved to handle cases where a large number of documents are selected. YASE tries to retrieve everything - which can take quite a while if the collection is large and the query selects a large majority of documents.

YASE Installation Guide

Platforms supported by YASE

YASE should work on most UNIX platforms. The author has tested it on SUSE Linux 8.0 only. YASE also works on Win32 platforms, including Windows 98. However, the installation process on Win32 is not documented. Visual C++ (6.0) project files are provided so that you can build YASE on Windows.

Pre-requisites for compiling YASE

YASE requires the GNU Make utility.

External libraries used by YASE

YASE can optionally use the libxml library to enable parsing of XML and HTML files. A version of this is included in the distribution.

YASE also uses the wget (version 1.5.3) utility for traversing web sites. A version of this tool is included in the distribution; it has been modified so that it can be linked to YASE as a library.

Building and Installing YASE on Linux

To build YASE follow these steps.

  • Download the latest version of YASE. See Downloading YASE for more information on how to obtain YASE.
  • Decide on a home directory for YASE. The default is /usr/local.
  • Unpack yase into a build directory as follows (this should be different from the home directory you decided above):
    mkdir [build_directory]
cd [build_directory]
tar xvfz [yase release file]
    or, if your tar command does not support the 'z' option:
    mkdir [build_directory]
cd [build_directory]
gunzip [yase release file]
tar xvf [yase tar file]
  • You should now end up with the YASE sub-directory. cd to this directory.
  • Execute configure as follows:
    ./configure --prefix=[yase home directory]
    You can specify '--enable-libxml' to enable linking of libxml. wget is linked in by default; you can disable linking of wget by specifying '--disable-wget'.
  • You can now build YASE by simply executing:

    make
  • To install YASE you can execute:
    make install
  • See Configuring and using YASE for information on how to execute YASE.
